# Robot Packages Directory

Place your robot description packages here.

## Adding Packages

### Method 1: Copy from Existing Workspace

```bash
cd ~/codes/ros2-moveit-workspace/src/
cp -r /path/to/your_robot_description .
```

### Method 2: Clone from Repository

```bash
cd ~/codes/ros2-moveit-workspace/src/
git clone https://github.com/your-org/your_robot_description.git
```

### Method 3: Symbolic Link

```bash
cd ~/codes/ros2-moveit-workspace/src/
ln -s /path/to/your_robot_description .
```

## Example: Ranger Garden Assistant

```bash
cd ~/codes/ros2-moveit-workspace/src/

# Copy robot description
cp -r ~/codes/ranger-garden-assistant/src/ranger_description .

# Copy arm packages (if separate)
cp -r ~/codes/ranger-garden-assistant/src/piper_ros .
```

## Package Requirements

Your robot package should include:

- **URDF/Xacro files**: Robot description with links and joints
- **Mesh files**: STL or DAE files for visualization
- **Launch files** (optional): For loading robot description
- **Package manifest**: `package.xml` and `CMakeLists.txt`

## Typical Structure

```
src/
├── your_robot_description/
│   ├── urdf/
│   │   ├── robot.urdf.xacro
│   │   └── ...
│   ├── meshes/
│   │   ├── visual/
│   │   └── collision/
│   ├── launch/
│   ├── config/
│   ├── package.xml
│   └── CMakeLists.txt
└── your_robot_moveit/        # Generated by Setup Assistant
    ├── config/
    ├── launch/
    ├── package.xml
    └── CMakeLists.txt
```

## After Adding Packages

1. **Install dependencies**:
   ```bash
   rosdep install --from-paths . --ignore-src -r -y
   ```

2. **Build the workspace**:
   ```bash
   cd /workspace
   colcon build --symlink-install
   source install/setup.bash
   ```

3. **Launch Setup Assistant**:
   ```bash
   ros2 launch moveit_setup_assistant setup_assistant.launch.py
   ```

## Notes

- **Don't commit build artifacts**: The `.gitignore` excludes `build/`, `install/`, and `log/`
- **Do commit generated configs**: Keep your `*_moveit` packages in version control
- **Dependencies**: If your robot depends on other packages, include them in `src/` as well
- **Submodules**: Consider using git submodules for packages from external repositories
